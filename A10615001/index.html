程式流程

運作原理

  • 鎖。鎖提供了互相排斥的功能，保證某個時間節點只有一個 CPU 能持有鎖。如果 xv6 只能在持有特定的鎖時才能使用數據結構，那麽這樣就可以保證同一時間只有一個 CPU 能使用這個數據結構。這樣，我們就稱這個鎖保護了數據結構。

  • xv6 用结构体 struct spinlock。该结构体中的关键成员是 locked 。这是一个字，在锁可以被获得时值为0，而当锁已经被获得时值为非零。

Acquire

  • acquire 一定要在可能獲得鎖的 xchg 之前調用 pushcli。如果兩者顛倒了，就可能在幾個時鍾周期裏，中斷仍被允許，而鎖也被獲得了，如果此時不幸地發生了中斷，系統就會死鎖。類似的，release 也一定要在釋放鎖的 xchg 之後調用 popcli。

  • acquire 会记录获得锁的 CPU 和栈信息，以便调试。当某个进程获得了锁却没有释放时，这些信息可以帮我们找到问题所在。当然这些信息也被锁保护着，只有在持有锁时才能修改。

1602 release(struct spinlock *lk) 
1603 { 
1604   if(!holding(lk)) 
1605     panic("release"); 
1606 
1607   lk−>pcs[0] = 0; 
1608   lk−>cpu = 0; 
1609 
1610   
1611   // 確保所有的儲存
1612   // 在釋放鎖之前，要看見其它關鍵的部分 
1613   //C編譯器和硬件都可以重新加載負載
1614   // stores; __sync_synchronize() tells them both not to. 
1615   __sync_synchronize(); 
1616 
1617   //釋放鎖。equivalent to lk−>locked = 0. 
1618   // 這個代碼不能使用C的賦值，因為它可能不是原子的。 一個真正的OS將在這裏使用C原子。  
1619   
1620   asm volatile("movl $0, %0" : "+m" (lk−>locked) : ); 
1621 
1622   popcli(); 
1623 }
