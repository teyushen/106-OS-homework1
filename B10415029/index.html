Chapter 4 －Lock

四資工三甲 B10415029 呂芯萍

Locking

為了防止CPU之間互相干擾，Lock提供了互斥功能，保證某個時間點只會有一個CPU持有Locking。xv6僅能在持有特定的鎖時才能使用數據結構，故能保證同一時間只有一個CPU能使用，並稱這個Locking保護了數據結構。

注意

  1. 讀代碼時，是否還有其他CPU正在執行此代碼
  2. 代碼可能由多條機器指令組成，另一CPU可能在這些指令中影響
  3. 不能假設代碼是順序執行的

Race conditions

xv6中的IDE磁盤驅動會維護一個未完成磁盤請求的linked list，若無併發請求，程式碼如下：


















struct list{
    int data;
    struct list *next;
};

struct list *list = 0;


void
insert(int data)
{
struct list *l;
l = malloc(sizeof *l);
l->data = data;
l->next = list;
list = l;
}

如果兩個CPU同時執行insert，可能會兩個都執行15行，而不去執行16行(Figure 4-1)，後執行的一個將會覆蓋前執行的一個，於是先賦值一個進程中添加的節點就丟失了，即為race condition(競爭條件)。

競爭問題在於CPU執行時間以及其內存操作的先後決定。故調試insert時加入輸出語句，以改變執行時間，進而解決競爭問題。

不過若使用Lock來避免競爭，只需加入幾行代碼(未標號的)，程式碼如下：















struct list *list = 0;
struct lock listlock;

void
insert(int data)
{
    struct list *l;
       acquire(&listlock);
    l = malloc(sizeof *l);
    l->data = data;
    l->next = list;
    list = l;
       release(&listlock);
}

Code: Locks

xv6使用struct spinlock。Lock可被獲得時值為0，而當Lock已被獲得時值為非0。以下程式碼呈現xv6如何獲得Lock：











void
acquire(struct spinlock *lk)
{
    for(;;) {
        if(!lk->locked) {
            lk->locked = 1;
            break;
        }
    }
}

此程式碼無法保證互斥，比如幾個CPU接連執行到5行發現lk->locked為0，然後都執行6、7行取得Lock。這時多個CPU同時持有Lock就違反了互斥。不過上述問題主要出在第5、6行是分開執行的，若要保證代碼正確，就必須讓第5、6行是同時執行。能夠使用xv6的一條特殊指令xchg達到同時執行。

xchg

  1. xchg將會交換內存中的一個字和一個暫存器的值。函數acquire在循環中反覆使用xchg，每次都讀取lk->locked，然後設置為1。
  2. 如果Lock已被持有，lk->locked便為1，故xchg會回傳1然後繼續循環。如果xchg回傳0，但是acquire已成功獲得Lock(即lk->locked由0變1)，循環即停止。
  3. 一旦有CPU獲得Lock，acquire會記錄獲得Lock的CPU和stack trace。
  4. 補：release則反之，用以清除調適訊息並釋放Lock。

Code: Using locks

需要Lock

  1. 當一個CPU在寫一個變量，而同時可能有另一個CPU讀或寫變量，此時須使用Lock防止兩操作重疊。
  2. 當一不變量持有Lock時，如果不變量涉及到多個數據結構，通常每個數據結構都需一個單獨的Lock保護起來，才能維持不變量。

不需要Lock

  1. 當效率不是很重要的時候，僅使用單一CPU即可。通常會使用內核Lock，進入內核時必須擁有Lock，退出內核時則釋放Lock。俗稱giant kernel lock(內核巨鎖)

Deadlock and lock ordering

如果一個代碼需要多個Lock，那麼代碼每次運行都要以相同的順序獲得Lock，否則就有死鎖的危險。

假設某段代碼的兩條執行路徑都需要鎖A和B，但路徑1獲得鎖的順序是A、B，路徑2的順序時B、A，當路徑一先獲得鎖A時，而在他繼續獲得鎖B前，路徑2恰好獲得鎖B，這樣就死鎖了，雙方景獲得一鎖且無法釋放擁有的鎖給對方，故皆無法執行
